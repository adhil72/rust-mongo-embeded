# Version 0.1.5

## Changes

- **Enhanced Progress Reporting**:
  - `start_with_progress` callback now receives an `InitStatus` enum instead of just `DownloadProgress`.
  - Added new status states: `CheckingDB`, `ValidatingInstallation`, `Downloading`, `DownloadProgress`, and `DBInitialized`.
  - This allows for more granular feedback during the MongoDB initialization process.

- **Refactoring & Fixes**:
  - Removed unused `db_path` field from `MongoProcess` struct to fix compiler warnings.

## Example Usage

```rust
use mongo_embedded::{MongoEmbedded, InitStatus};

async fn start_mongo() -> anyhow::Result<()> {
    let mongo = MongoEmbedded::new("7.0.2")?;

    mongo.start_with_progress(|status| {
        match status {
            InitStatus::CheckingDB => println!("Checking database..."),
            InitStatus::ValidatingInstallation => println!("Validating installation..."),
            InitStatus::Downloading => println!("Downloading MongoDB..."),
            InitStatus::DownloadProgress(progress) => {
                if let Some(pct) = progress.percentage {
                    println!("Download progress: {:.2}%", pct);
                }
            },
            InitStatus::DBInitialized => println!("Database initialized!"),
        }
    }).await?;
    
    Ok(())
}
```
